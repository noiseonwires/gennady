**Бот для модерации и вовлечения сообществ в Telegram на базе ИИ.**

Геннадий следит за вашим чатом, понимает, что говорят люди, и действует
по *вашим* правилам - а не по фиксированному универсальному фильтру. Опишите правило модерации
обычным языком, и бот будет его применять: спам, оскорбления, оффтоп, NSFW, мошенничество -
всё, что требуется вашему сообществу. Он читает и текст, *и* изображения, выполняет
настроенные вами действия, поддерживает жизнь в чате сводками и приветствиями и поставляется
с полноценным веб‑интерфейсом, так что вам не придётся открывать файл конфигурации,
если вы сами этого не захотите.

Написан на Go. Запускается как один статический бинарник, Docker-контейнер или облачный сервис.
Использует локальный SQLite или удалённую базу в облаке. Свободный проект с открытым исходным кодом под лицензией AGPL-3.

> **Проверен в боевых условиях.** Геннадий работает с августа 2025 года в Telegram-чате
> с более чем 1000 участников, а с марта 2026 года модерирует чат комментариев одного
> крупного чешского новостного канала в Telegram.

## Зачем нужен Геннадий?

Большинство ботов-модераторов предлагают выбрать правила из готового списка. Геннадий позволяет
просто *описать* нужное поведение обычным языком, а затем привязать вердикты ИИ к нужным
действиям. Поэтому один и тот же бот может вести строгий канал юридических новостей и спокойную
группу по хобби - достаточно поменять промпт.

## Возможности

### 🛡️ ИИ-модерация на ваших условиях

- **Промпты модерации под ваш чат.** Опишите обычным языком, что считать нарушением. Настройте
  бота на спам, оскорбления, оффтоп, NSFW, мошенничество или любые правила, важные именно для
  вашего сообщества. Правила чата попадают в промпт, поэтому ИИ проверяет сообщения по *вашим*
  меркам.
- **Действия для каждого вердикта.** Настройте, что делать с каждым вердиктом ИИ (например,
  `spam`, `profanity`, `nsfw`, `unsure`). На один триггер можно повесить сразу несколько
  действий.
- **Гибкие действия:** автоудаление, автомут на заданное время или навсегда, автопредупреждение
  или отчёт в админ-чат, где решение примет человек. Всё настраивается отдельно для каждого
  правила.
- **Предупреждения от ИИ.** Когда пользователь переходит черту, бот может написать личное,
  контекстное предупреждение вместо сухого шаблона.
- **Быстрая и сильная модели ИИ.** Массовые недорогие проверки можно отправлять на быструю
  модель, а более сильную оставлять для тонких задач. Настраивается отдельно для каждой функции,
  с автоматическим переключением по длине текста.
- **Профили с учётом репутации.** Если включить ИИ-профили пользователей, модератор будет
  оценивать сообщение с учётом известного стиля и истории автора. Это помогает реже ошибаться
  на постоянных участниках и строже смотреть на тех, кто нарушает не впервые.
- **Исключения для админов.** При желании модерацию можно отключить для администраторов и списка
  доверенных пользователей.

### 🖼️ Безопасность не только в тексте

- **Модерация изображений через OCR и компьютерное зрение.** Нарушения, спрятанные в скриншотах
  и картинках, тоже не пройдут. Запустите OCR (Azure Vision или облачный OCR.space),
  чтобы достать текст из изображения и проверить его теми же правилами модерации.
- **Azure Vision и Content Safety** для анализа изображений и оценки безопасности.
- **Проверка фото профиля.** При желании бот проверит фото профиля нового пользователя через
  Azure Content Safety на первом сообщении. Если фото подозрительное, в профиль добавится
  пометка, которую модератор учтёт дальше.

### 👮 Инструменты модерации

- **Система мутов** с готовыми длительностями и автоматическим снятием. Если нужно, мут можно
  применить сразу во всех модерируемых чатах.
- **«Жестокий мут»** для повторных нарушителей: пользователя **не** ограничивают официальными
  средствами Telegram (в аккаунте не видно мута, он может писать и отправлять сообщения), но
  каждое его сообщение тихо удаляется сразу после отправки. Без уведомления «вы в муте» и без
  ошибок упорный нарушитель продолжает говорить в пустоту - обычно это раздражает гораздо
  сильнее обычного мута.
- **Предупреждения** с полной историей по каждому пользователю.
- **Ручные действия** из админ-чата через встроенную клавиатуру или из веб-интерфейса.
- **Автоудаление по темам.** Telegram умеет включать автоудаление только для всего чата сразу;
  бот удаляет сообщения только в выбранных вами темах или тредах, с заданным временем жизни.
  Так можно держать тему с объявлениями чистой и не трогать основной чат.
- **Очистка закреплённых сообщений и служебных уведомлений.**
- **Журнал действий** по каждому муту, размуту, предупреждению и удалению - видно, кто выполнил
  действие: бот или админ.

### 👤 Что известно о пользователях

- **Профили пользователей от ИИ.** Бот каждый день собирает поведенческие профили с оценкой
  репутации и использует их для более точных решений по модерации.
- **Отслеживание пользователей.** История смены `@username` и отображаемого имени, время первого
  появления в каждом чате и графики активности по дням.
- **Поиск фейков и подмены.** Бот предупредит, если новый аккаунт начнёт использовать
  `@username`, который раньше принадлежал другому аккаунту («удалился и зарегистрировался заново
  с тем же ником»).

### 💬 Для жизни сообщества

- **Утренние приветствия.** Ежедневное доброе утро с погодой, праздниками и историческими
  событиями «в этот день» - сгенерированное ИИ или собранное только из данных.
- **Ежедневные сводки.** ИИ кратко пересказывает, что обсуждали в чате за день.
- **Сводки сообщений.** Длинные сообщения автоматически получают короткое резюме.
- **Сводки ссылок.** Когда кто-то делится ссылкой, бот загружает страницу и публикует краткое
  содержание. Ещё он может перевести страницу на язык вашего чата и использует цепочку запасных
  провайдеров, если страницу сложно разобрать.
- **Креативные ответы.** Остроумные ответы ИИ по контексту, с ограничением частоты, чтобы бот
  оставался приятным собеседником, а не спамил.

### 📰 Публикация RSS

- **RSS-ленты по расписанию.** Публикуйте новые записи из любой RSS-ленты по отдельному
  расписанию для каждой ленты.
- **Перевод и резюме от ИИ.** При желании бот переведёт записи на язык вашего чата и/или кратко
  перескажет их перед публикацией.

### 🖥️ Веб-интерфейс

Есть встроенная админ-панель, которую можно включить по желанию. Она защищается паролем и
Telegram OTP и позволяет управлять ботом без SSH:

- **Наглядная настройка** всех параметров, ИИ-моделей, правил модерации и RSS-лент.
- **Диагностика:** проверка внешних API, просмотр готовых промптов модерации, статус вебхука,
  живые логи, статистика базы данных и расход ИИ-токенов по сервисам.
- **Модерация из браузера:** мут, размут, предупреждения, просмотр сообщений (фильтр по
  чату/пользователю, переходы по цепочкам ответов) и профилей пользователей.
- **Перенос настроек и данных:** скачать или загрузить конфиг, переменные окружения или базу
  SQLite; перенести конфиг из файла в базу данных и обратно.
- **Перезапуск в один клик.**

### ⚙️ Запуск и данные

- **Запускается где угодно.** Один статический бинарник Go, systemd-сервис, Docker / Docker
  Compose или облачный контейнер.
- **Long-polling или вебхук.** Для локального запуска подойдёт простой long-polling, для облака
  и serverless-развёртываний - режим вебхука.
- **SQLite локально или удалённая база.** Храните всё в локальном SQLite-файле или подключите
  удалённую libSQL/Turso-базу. Это удобно для временных облачных контейнеров вроде Bunny.net
  Magic Containers.
- **Настройка без файлов.** Можно поднять минимальный облачный экземпляр (например, на Bunny.net)
  без подключённого файла конфигурации, а потом настроить всё через веб-интерфейс. Бот сам
  создаст и сохранит конфигурацию в базе данных.
- **Интерфейс и документация** по конфигурации доступны на русском и английском.
- **Безопасно для нескольких экземпляров.** Блокировки на уровне базы не дают расписанию
  сработать дважды, если в облаке одновременно работает больше одного экземпляра - например, во
  время передеплоя, когда старый и новый контейнеры ненадолго пересекаются.

## Внешние сервисы

Gennady сам справляется с базовой модерацией, а внешние сервисы подключает только для
дополнительных возможностей. Всё ниже необязательно и настраивается отдельно.

| Сервис | Для чего используется | Примечания |
|---|---|---|
| **Azure OpenAI** | Все функции ИИ | Или любой эндпоинт, совместимый с OpenAI. |
| **OpenAI / OpenAI-совместимые** | Все функции ИИ | OpenAI, OpenRouter, Groq, LiteLLM, локальный Ollama и т. д. |
| **Azure Vision** | Анализ изображений и OCR | Для модерации изображений. |
| **Azure Content Safety** | Оценка безопасности контента | Необязательная дополнительная проверка. |
| **OCR.space** | Извлечение текста из изображений | Облачный OCR API; есть бесплатный тариф, свой сервер не нужен. |
| **Diffbot** | Извлечение содержимого ссылок | Первый вариант в цепочке извлечения ссылок. |
| **ExtractorAPI** | Извлечение содержимого ссылок | Запасной экстрактор. |
| **Cloudflare Browser Rendering** | Извлечение содержимого ссылок | Рендерит страницы, где много JavaScript. |
| **Open-Meteo** | Погода в утренних приветствиях | Бесплатно, без API-ключа. |
| **OpenHolidays API** | Праздники в приветствиях | Бесплатно, без API-ключа. |
| **Wikipedia (On This Day)** | Исторические события в приветствиях | Бесплатно, без API-ключа. |
| **Turso / BunnyNet / libSQL** | Удалённая база данных | Для облачных развёртываний. |

## Быстрый старт

```bash
# 1. Скачайте код
git clone https://github.com/noiseonwires/gennady.git
cd gennady

# 2. Создайте конфиг
cp config.example.yaml config.yaml
#    отредактируйте config.yaml - задайте bot_token, admin.chat_id, moderation.chat_id и ключи ИИ

# 3. Соберите и запустите
go build -o gennadium .
./gennadium
```

Минимум нужен **токен Telegram-бота** от [@BotFather](https://t.me/BotFather). Ключи для ИИ
необязательны, но без них функции ИИ не заработают.

➡️ Полные инструкции для локального запуска, Docker и облака - в
[installation_ru.md](docs/installation_ru.md).

➡️ Все параметры конфигурации описаны в [config_ru.md](docs/config_ru.md), а полные
автогенерируемые справочники лежат в [CONFIG_REFERENCE_ru.md](docs/CONFIG_REFERENCE_ru.md) и
[CONFIG_REFERENCE_en.md](docs/CONFIG_REFERENCE_en.md).

## Флаги командной строки

| Флаг | Описание |
|---|---|
| `-config <путь>` | Путь к YAML-файлу конфигурации (по умолчанию `config.yaml`). |
